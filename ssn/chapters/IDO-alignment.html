IDO Alignment Module

This section introduces the alignment of SOSA to Industrial Data Ontology (IDO), which is standardized as ISO 23726-3. This serves to axiomatically clarify the intended meaning of SOSA terms in terms of IDO and enable usage of SOSA/SSN with IDO-aligned ontologies.

An RDF file containing a graph corresponding to this alignment is available.

Namespaces

The following namespace prefixes are used in the alignment of SOSA to IDO.

sosa:
http://www.w3.org/ns/sosa/
lis:
http://rds.posccaesar.org/ontology/lis14/rdl/

Class Alignments

The upper-level classes from SOSA are aligned with the IDO classes as follows. The alignment of subclasses of these upper-level classes follows logically from these correspondences.

The expression C1 ⊑ C2 represents a SubClassOf axiom.

The definition of each IDO concept is displayed as a tooltip when hovering over the concept.

  • sosa:ActuatableProperty ⊑ lis:Disposition
  • sosa:Actuator ⊑ lis:PhysicalObject
  • sosa:Deployment ⊑ lis:Activity
  • sosa:Execution ⊑ lis:Activity
  • sosa:ExecutionCollection ⊑ lis:Activity
  • sosa:FeatureOfInterest ⊑ lis:Object
  • sosa:ObservableProperty ⊑ lis:Quality
  • sosa:Platform ⊑ lis:PhysicalObject
  • sosa:Procedure ⊑ lis:InformationObject
  • sosa:Property ⊑ lis:Dependent
  • sosa:Result ⊑ lis:Object
  • sosa:SampleCollection ⊑ lis:Object
  • sosa:Sampler ⊑ lis:PhysicalObject
  • sosa:Sensor ⊑ lis:PhysicalObject
  • sosa:Stimulus ⊑ lis:Activity
  • sosa:System ⊑ lis:System

Property Alignments

The upper-level object properties from SOSA are aligned with the IDO properties as follows. The alignment of subproperties of these upper-level properties follows logically from these correspondences.

The expression P1 ⊑ P2 represents a SubPropertyOf axiom.

The definition of each IDO property is displayed as a tooltip when hovering over the concept.

  • sosa:actsOnProperty ⊑ lis:realizes
  • sosa:deployedAsset ⊑ lis:hasPassiveParticipant
  • sosa:hasFeatureOfInterest ⊑ lis:hasParticipant
  • sosa:hasInput ⊑ lis:hasPassiveParticipant
  • sosa:hasOutput ⊑ lis:hasPassiveParticipant
  • sosa:hasResult ⊑ lis:hasPassiveParticipant
  • sosa:hasSubSystem ⊑ lis:hasFunctionalPart
  • sosa:hosts ⊑ lis:locatedRelativeTo
  • sosa:implements ⊑ lis:concretizes
  • sosa:madeBySystem ⊑ lis:hasActiveParticipant
  • sosa:madeExecution ⊑ lis:activeParticipantIn
  • sosa:phenomenonOccurred ⊑ lis:temporalExtentOf
  • sosa:phenomenonTime ⊑ lis:hasTemporalExtent
  • sosa:usedProcedure ⊑ lis:hasParticipant
