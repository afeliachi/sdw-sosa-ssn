Actuating Procedure — A workflow, protocol, plan, algorithm, or computational method specifying how to make an Actuation. An Actuating Procedure is re-usable, and might be applied in many Actuations. It explains the steps to be carried out to arrive at reproducible results.

sosa:Actuation

IRI: http://www.w3.org/ns/sosa/Actuation

an OWL Class

Actuation — Act of carrying out an actuating sosa:Procedure using an sosa:Actuator to change the value of an actuatable sosa:Property of a sosa:FeatureOfInterest.

In this context

  • sosa:hasResult SHALL indicate the expected value of the sosa:Property at the completion of the sosa:Actuation
  • sosa:relatedObservation SHOULD be used to link to an associated sosa:Observation or sosa:ObservationCollection that monitors the actuated property in order to verify that the result value has been obtained

An sosa:Actuation concerns a single sosa:Property of a single sosa:FeatureOfInterest. Changes to multiple properties, or other combinations of Actuations, may be described in an sosa:ActuationCollection.

Alternatively, applications may choose to encapsulate complexity by defining a complex property with multiple individual components, and a corresponding complex result (e.g., a vector). This approach is not prohibited by the SSN Ontology, but the details are beyond the scope of SSN.

The different time-properties on a sosa:Actuation support the description of plans, forecasts and predictions as well as descriptions of various historical scenarios. See Temporal properties for patterns related to these.

Example The activity of automatically closing a window if the temperature in a room drops below 20 degree Celsius. The activity is the sosa:Actuation and the device that closes the window is the sosa:Actuator. The sosa:Procedure is the rule, plan, or specification that may define the condition that triggers the sosa:Actuation (a drop in temperature).

sosa:ActuationCollection

IRI: http://www.w3.org/ns/sosa/ActuationCollection

an OWL Class

ActuationCollection added in 2023 Edition. This term is non-normative, pending further implementation experience

Actuation Collection — Collection (set) of one or more sosa:Actuations or sosa:ActuationCollections, i.e., collections may be nested.

The following sosa:Actuation properties may appear in an sosa:ActuationCollection — sosa:hasFeatureOfInterest, sosa:usedProcedure, sosa:madeByActuator, sosa:actsOnProperty, sosa:endTime, sosa:hasResult, sosa:hasSimpleResult. If it is present, the value of a property summarizes the values of the matching properties of the member actuations, where membership is either direct or transitive through one or more member actuation collections.

The following consistency rules apply to the Actuation properties listed above:

  1. Where an individual sosa:ActuationCollection omits a property, a member sosa:Actuation (direct or transitive) MAY have any value for that property.
  2. Where an individual sosa:ActuationCollection has a single value for a property, each member sosa:Actuation (direct or transitive) MUST have that same value for that property — i.e., the collection is homogeneous in that property. That property MAY then be omitted in any member sosa:Actuation or sosa:ActuationCollection.
  3. Where an individual sosa:ActuationCollection has more than one value for a property, each member sosa:Actuation (direct or transitive) MUST have a value for that property that matches one of the values for the property in the collection.
  4. Where an individual sosa:ActuationCollection has a value for a property that is a range or interval, each member sosa:Actuation (direct or transitive) MUST have a value for that property that matches or falls within that range or interval.
  5. Where an individual sosa:ActuationCollection has more than one value for a property that is a range or interval, each member sosa:Actuation (direct or transitive) MUST have a value for that property that matches or falls within one of those ranges or intervals.

    6. The members of a collection do not necessarily share a common value for any property.

    Examples of Collections conforming to the rules are provided in and Collections and their members' property values. These illustrate all the cases described above.

sosa:Actuator

IRI: http://www.w3.org/ns/sosa/Actuator

an OWL Class

Actuator — An entity that implements an actuating sosa:Procedure to change the value of a actuatable sosa:Property. This may be a device, a piece of infrastructure, an agent (including humans), or software (simulation).

An Actuator may have a geographic location. See Location and Geometry for patterns to describe this.

Scope note Specific actuator types may be implemented as sub-classes of this class or its sub-classes. Their descriptions may capture the properties and characteristics of a whole class of actuators, corresponding with the information typically found in a spec-sheet or data-sheet.

Individual actuator instances are members of this class or its sub-classes.
Example A window actuator for automatic window control, i.e., opening or closing the window.
Sub class of sosa:System
Restrictions sosa:madeActuation ONLY ( sosa:Actuation OR sosa:ActuationCollection )
sosa:actsOn ONLY sosa:Property
sosa:implements ONLY sosa:ActuatingProcedure
