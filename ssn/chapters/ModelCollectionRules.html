Execution collections — consistency rules

Properties of the members of ExecutionCollections must follow the consistency rules provided in their definitions. The following examples illustrate each case from the rules, in the context of an ObservationCollection. The patterns shown also apply to the other collection types.

Rule 1 — absent property on the collection

Where an individual ExecutionCollection omits a property, a member Execution (direct or transitive) MAY have any value for that property.

Properties that are specified on the collection apply to every member, i.e., in this example sosa:observedProperty, sosa:madeBySensor, qudt:hasUnit. Properties that are not specified for the collection are provided individually on each member, i.e., sosa:hasFeatureOfInterest, sosa:phenomenonTime, sosa:hasSimpleResult. 




Rule 2 — single property value on the collection




    Where an individual ExecutionCollection has
    a single value for a property, each member Execution (direct or transitive)
    MUST have that same value for that property — i.e., the collection (set) is
    homogeneous in that property.
    That property MAY then be omitted in any member Execution or
    ExecutionCollection.





    The collection has a single value for sosa:observedProperty, and any member with that property has the
    same value.







Rule 3 — multiple property values on the collection




    Where an individual ExecutionCollection has
    more than one value for a property, each member Execution
    (direct or transitive) MUST have a value for that property that matches one of the values for the
    property in the collection.





    The collection has a multiple values for sosa:hasFeatureOfInterest, and each member has one of these
    values for that property.







Rule 4 — range property value on the collection




    Where an individual ExecutionCollection
    has a value for a property that is a range or interval, each member
    Execution (direct or transitive) MUST have a value for
    that property that matches or falls within that range or interval.





    The collection has a range of values for sosa:phenomenonTime, and the value for that property on each
    member falls within the range.







Rule 5 — multiple ranges property value on the collection




    Where an individual ExecutionCollection
    has more than one value for a property that is a range or interval, each member
    Execution (direct or transitive) MUST have a value for
    that property that matches or falls within one of those ranges or intervals.





    The collection has two ranges of values for sosa:phenomenonTime, and the value for that property on
    each member falls within one of the ranges.
