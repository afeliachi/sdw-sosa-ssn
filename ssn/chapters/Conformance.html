Data conforms to SSN if:

  • Access to data is organized around Actuations, Observations, and Samplings, which represent executions of Actuating, Observing and Sampling Procedures, implemented by Actuators, Sensors and Samplers.
  • The contents of all data fields are expressed using the appropriate RDF classes and properties from SSN. Where no such class or property exists, sub-classes or sub-properties specializing SSN in a domain-specific web ontology are employed.
  • All classes and properties from SSN are used in a way consistent with the semantics declared in this specification, as so any SSN sub-classes or sub-properties specialised in domain-specific ontologies used to express the concerned data.

SSN-compliant knowledge graphs MAY include additional non-SSN class instances, object and data properties, and additional RDF data.
